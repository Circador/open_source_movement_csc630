git cheatsheet

Resources Assignments git cheatsheet
  • git init
  • git status
  • git add <filename>
  • git rm <filename>
  • git reset <filename>
  • git commit: Joins two or more histories of the code together (i.e. If two individuals are adding to the same development, to bring both edits back into one file, the merge command will update all edits into one branch.) Although, if there are issues with edit overlaps, formely known as a merge conflict, the user will need to fix the overlapping edits before the merge is complete.
  • git diff
  • git log
  • git checkout: Change the workspace (i.e. the current files in the directory) to those located at the specified commit. In essence, moves "you" (a.k.a. HEAD) to the commit. If you add the -b flag, it will create a new branch with the specified name: git checkout -b <new_branch_name>.
  • git branch <branchname>
  • git remote
  • git push
  • git merge <branchname>: Fetch branches and all other objects from one or more repositories. Note: fetch does not merge or overwrite local files.
  • git fetch
  • git pull <remote_repo> <remote_branch>: Runs git fetch (with the given parameters) followed by git merge to merge retrieved branch heads into the current branch (using --rebase runs git rebase instead of git merge). Fetches and replays changes from remote branch since it has diverged from the local branch.

    If remote changes overlap with local uncommitted changes, the merge will be automatically canceled.
