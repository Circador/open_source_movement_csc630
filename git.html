git cheatsheet

Resources Assignments git cheatsheet
  • git init
  • git status: Tells you the tree and index status (i.e. whether you're ready to commit or not)
  • git add <filename>
  • git rm <filename>: Remove a file from git's index, and remove it from the working tree if it exists there. git rm cannot be used to only remove a file from the working tree. rm <filename> (without git) removes the file from only the working tree.
  • git reset <filename>
  • git commit: Saves the latest staged changes into your local repository. You can add a message with your commit describing your changes. Optional flag includes the -b flag which allows you to add a string message with the command. Internally, it also logs a commit object which includes information about the commit.
  • git diff: Shows changes between commits, commit and working tree etc: git diff [options] [<commit>] [--] [<path>…​]
  • git log: Using git log shows the commit history of the user. The command will show the author of the commit, the date of the commit, and a commit hash that you can use a git checkout command on to view a detached head version of the repo.
  • git checkout: Change the workspace (i.e. the current files in the directory) to those located at the specified commit. In essence, moves "you" (a.k.a. HEAD) to the commit. If you add the -b flag, it will create a new branch with the specified name: git checkout -b <new_branch_name>.
  • git branch <branchname>

    This command will create a new branch in the current repository, with the name 'branchname'. Without the option branchname parameter, it will list all branches.

  • git remote: Remotes are repositories other than the one on your local disk which you can push your changes into or pull from. Use git remote to manage these repositories. With no arguments, git remote will show a list of the existing remotes. To perform operations on these remotes, use subcommands like add, remove, rename, etc...
  • git push: Updates the references (HEAD, master, etc.) on the remote using local references. It sends objects the are present locally but not remotely in order to bring the remote up to date with your local copy.
  • git merge <branchname>
  • git fetch
  • git pull <remote_repo> <remote_branch>
